DataTypec back to ToC or Class ToC

IRI: http://ocean-data.org/schema/data-type/v1.0/DataType

A data type can include more meaning beyond the syntax, such as who created the data type, the source standard that the data type derives from, the operations that can be done on datasets of that data type, and typical scientific domains, software programs and/or instruments that use the data type, etc. The intent is that any humans or machines that when they encounter that data type can quickly 'understand' or be in a situation to at least 'process' details within the dataset without even downloading that dataset.

InferredParamterc back to ToC or Class ToC

IRI: http://ocean-data.org/schema/data-type/v1.0/InferredParamter

An inferred parameter is a parameter whoa value is not explicitly recorded in the Dataset, but that can be inferred based on its data type and related metadata. For example, dataset collected on a Cruise might not explicitly record the cruise in the dataset, but that it can be inferred through the metadata: (odo:Dataset -> odo:fromDeployment [ dc:title ]).
has super-classes
parameter c

dataValueFormatdp back to ToC or Data Property ToC

IRI: http://ocean-data.org/schema/data-type/v1.0/dataValueFormat

A string, indicating a format for the data value that enables correct processing of the value. For example, a date value may have a format such as 'YYYY-MM-DD'. Since the a format specified here are implementation specific, this definition does not enforce any particular range on this property.
